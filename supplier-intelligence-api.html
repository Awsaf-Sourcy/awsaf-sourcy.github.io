Structured Output Supplier Matching Integration Code Strategy Guide

Ask any question about suppliers, products, or sourcing capabilities. The API retrieves from WeChat chats, TMAPI product tags, and the supplier database, then generates answers via one of 4 query types. Embed a JSON schema in your question to get structured output. Only question is required — query_type and node_name have sensible defaults.

Natural Language Structured Output Supplier Matching No Auth Required
RAG_COMPLETION Fast

Standard retrieval + LLM completion. ~3–5s. Best for most queries: lookups, requirement matching, chatbot responses.

GRAPH_COMPLETION_CONTEXT_EXTENSION Slow

Knowledge graph traversal + extended context. ~60–90s. Use for deep comparisons, market overviews, and multi-hop reasoning.

GRAPH_COMPLETION Alt

Graph retrieval without context extension. Faster than CONTEXT_EXTENSION but less thorough.

GRAPH_COMPLETION_COT Alt

Graph retrieval with chain-of-thought reasoning. Exposes the thinking process but slower.

Structured Output Pattern

      

        
Embed a JSON schema in question. The LLM conforms its output to match.

        
[Your question]. Output your response using this structure: {
  [JSON schema]
} [Optional: "Focus on being actionable"]

        
Tips

        
    
          
  • Keep schemas flat. Use descriptive field names.
    • 
          
  • Include example values: "confidence": "high/medium/low"
    • 
          
  • Add partial_matches and no_results_reason for graceful fallback
    • 
          
  • Use explicit rules to prevent hallucination
    • 
        

      

      
Advanced: Rigorous Prompt with Evidence & Partial Matches

      

        
GPT-improved prompt: separates qualified from partial matches, requires evidence citations, confidence scoring, and a no_results_reason fallback.

        
Find suppliers that can produce custom perfume bottle caps
using zinc alloy AND electroplating finish.
Return ONLY valid JSON matching this schema exactly
(no markdown, no extra keys):
{
  "qualified_suppliers": [{
    "supplier": "string", "material_capability": "string",
    "finish_options": ["string"], "moq": "string",
    "price_range": "string", "reasoning": "string",
    "evidence": ["string"], "confidence": "high|medium|low"
  }],
  "partial_matches": [{
    "supplier": "string", "matched": ["string"],
    "missing": ["string"], "reasoning": "string",
    "evidence": ["string"], "confidence": "high|medium|low"
  }],
  "no_results_reason": "string"
}

Rules:
- qualified ONLY if BOTH zinc alloy AND electroplating explicit
- partial_matches for one-but-not-both (always include, [] if none)
- evidence = short supporting snippets (no fabrication, [] if none)
- Keep reasoning concise and tied to evidence

        

          
TechniqueWhy

          
            
qualified vs partial_matchesSurfaces near-misses even without perfect matches

            
evidencePrevents hallucination by requiring KB citations

            
confidenceMakes uncertainty explicit for downstream filtering

            
no_results_reasonGraceful fallback explaining why, not just empty

          
        

      


      
Supplier Matching from Requirements

      

        
Core use case: submit requirements, get ranked matches with gap analysis.

        
1
Define requirements (product, material, MOQ, budget, certs)

        
2
Embed requirements + output schema in question

        
3
API retrieves matching supplier data

        
4
LLM generates ranked matches with met/unmet analysis

        
Pro tip: RAG_COMPLETION for initial matching, then GRAPH_COMPLETION_CONTEXT_EXTENSION on top candidates.

      


      
Integration Code

      

        

          
Node.js

          
const res = await fetch(
  'https://labs.sourcy.ai/api/supplier-intelligence/ask',
  {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({
      question: 'Find umbrella suppliers',
      query_type: 'RAG_COMPLETION',
      node_name: 'supplier_intelligence'
    })
  }
);
const data = await res.json();

        

        

          
Python

          
import requests

resp = requests.post(
    "https://labs.sourcy.ai/api/"
    "supplier-intelligence/ask",
    json={
        "question": "Find umbrella suppliers",
        "query_type": "RAG_COMPLETION",
        "node_name": "supplier_intelligence"
    }
)
data = resp.json()
