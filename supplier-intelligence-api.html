1. Overview

The Supplier Intelligence API provides a natural-language Q&A interface over Sourcy's supplier knowledge base. It retrieves and synthesizes information from multiple sources:

WeChat Chat Logs

Real sourcing conversations — pricing, MOQ, customization details

TMAPI Product Tags

Product catalog metadata — categories, tags, supplier mappings

Supplier Table

Internal supplier database — company info, certifications

Growth Team Data

Collated chats and text dumps from growth team interactions

2. Endpoint & Parameters

POST https://labs.sourcy.ai/api/supplier-intelligence/ask
ParameterTypeRequiredDefaultDescription
questionstring Yes— Natural language question. Supports free-form, structured output, and multi-requirement matching.
query_typestringNo RAG_COMPLETION Retrieval strategy. See Query Types below.
node_typestringNo supplier_intelligence Knowledge base segment. Keep as supplier_intelligence.
Response format: {"answer": ["..."]} — The API returns a JSON object with an answer array containing one or more strings.

3. Query Types

RAG_COMPLETION
Fast Recommended

Standard RAG retrieval + LLM completion. Best for most use cases.

GRAPH_COMPLETION_CONTEXT_EXTENSION
Slow Recommended

Knowledge graph traversal + extended context. Most comprehensive results.

GRAPH_COMPLETION
Fast Alternative

Graph-based retrieval without context extension.

GRAPH_COMPLETION_COT
Slow Alternative

Graph retrieval with chain-of-thought reasoning.

8. Structured Output Pattern

    

      
No native structured mode. Embed a JSON schema directly in question and the LLM conforms its output.

      
Template

      
[Your question]. Output your response using this structure: {
  [your JSON schema with field descriptions]
} [Optional: "Focus on being actionable", "Be concise"]

      
Tips

      
    
        
  • Keep schemas flat and simple
    • 
        
  • Use descriptive field names
    • 
        
  • Include example values: "confidence": "high/medium/low"
    • 
        
  • Add a post-schema instruction for tone
    • 
        
  • For complex schemas, break into multiple calls
    • 
      

    


    
    
9. Supplier Matching from Requirements

    

      
Core use case: submit requirements, get ranked supplier matches with gap analysis.

      
1
Define sourcing requirements (product, material, MOQ, budget, certs)

      
2
Embed requirements + output schema in question

      
3
API retrieves matching supplier data from knowledge base

      
4
LLM generates ranked matches with met/unmet analysis

      
Generic Template

      
Find suppliers matching these sourcing requirements:
  Product: [product description]
  Material: [material requirements]
  Finish/Process: [surface treatment or process]
  MOQ: [quantity constraint]
  Budget: [price range per unit]
  Certifications: [required certifications]

Output as: {
  "supplier_matches": [{
    "rank": "number",
    "supplier_name": "string",
    "match_score": "percentage",
    "requirements_met": ["string"],
    "requirements_unconfirmed": ["string"],
    "requirements_not_met": ["string"],
    "estimated_unit_price": "string",
    "estimated_moq": "string",
    "notes": "string"
  }]
}

      

        Pro tip: Use RAG_COMPLETION for initial matching, then GRAPH_COMPLETION_CONTEXT_EXTENSION on top candidates for deeper verification.
      

    


    
    
10. Integration Code

    

      

        
Node.js / TypeScript

        
async function querySupplierIntelligence(
  question,
  queryType = 'RAG_COMPLETION'
) {
  const res = await fetch(
    'https://labs.sourcy.ai/api/supplier-intelligence/ask',
    {
      method: 'POST',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify({
        question,
        query_type: queryType,
        node_type: 'supplier_intelligence'
      })
    }
  );
  if (!res.ok) throw new Error(`${res.status}`);
  return res.json();
}

// Usage
const result = await querySupplierIntelligence(
  'Find umbrella suppliers with logo printing. ' +
  'Output as JSON: {"suppliers": [{"name": "str"}]}'
);

      

      

        
Python

        
import requests

def query_supplier_intelligence(
    question,
    query_type="RAG_COMPLETION"
):
    resp = requests.post(
        "https://labs.sourcy.ai/api/"
        "supplier-intelligence/ask",
        json={
            "question": question,
            "query_type": query_type,
            "node_type": "supplier_intelligence"
        }
    )
    resp.raise_for_status()
    return resp.json()

# Usage
result = query_supplier_intelligence(
    'Find zinc alloy perfume cap suppliers '
    'under MOQ 1000. Output as: '
    '{"suppliers": [{"name": "str"}]}'
)
